## 技術的な制約と既知の問題点

このプロジェクトは、APIを使用せずにWebブラウザを自動操作するという簡易的なアプローチを採用しているため、いくつかの技術的な制約と潜在的な問題点があります。

### 1. 自動操作の脆弱性 (Fragile Automation)
- キーボード操作 (`Ctrl+V`, `Enter` など) とクリップボード操作をシミュレートして入力を行っています。そのため、動作中にフォーカスが外れたり、システムの負荷が高い場合に正しく動作しない可能性があります。
- `Task.Delay` による固定待機時間が使用されているため、環境によっては処理が遅すぎたり、逆に早すぎて失敗することがあります。

### 2. クリップボードへの干渉
- データをブラウザに渡すためにクリップボードを使用します。元のクリップボードの内容を復元する処理は含まれていますが、動作中にユーザーがクリップボード操作を行うと競合する可能性があります。
- クリップボード履歴管理ソフトとの相性問題が発生する可能性があります。

### 3. 外部実行ファイルへの依存
- `ClipboardHistMngr.exe` や `HmFocusEachBrowserInputField.exe` などの外部ツールに依存しており、これらが存在しない場合やバージョンが不整合の場合に動作しません。
- 共有ライブラリ `HmConvAIWeb.js` を利用しており、これの更新が必要になる場合があります。

### 4. エラーハンドリングの不足
- C# や JavaScript のコード内において、例外処理でエラーが握りつぶされている（空の catch ブロックなど）箇所があり、問題発生時に原因の特定が困難な場合があります。

### 5. 秀丸エディタのバージョン依存
- マクロファイル内で `hidemaruversion "9.50.04";` と指定されており、特定のバージョン以降の秀丸エディタが必要です。

### 6. 結果の自動処理不可
- Web版のインターフェースを操作するだけであるため、AIからの応答をプログラムで取得・加工することはできません。あくまで「入力補助」に留まります。
